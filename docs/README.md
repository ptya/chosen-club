# Chosen Club Documentation

Welcome to the Chosen Club documentation. This directory contains all the essential documentation for understanding, developing, and deploying the application.

## Documentation Index

### [SPECIFICATION.md](./SPECIFICATION.md)
Complete application specification including:
- Overview and core features
- Game modes (Solo and Pair)
- Phase system (Phase 1, Phase 2, Phase 3+, Battle of the Chosen)
- User interface features
- Technical requirements
- Configuration options
- Edge cases and success criteria

**Read this first** to understand what the application does and how it works.

### [SCHEMA.md](./SCHEMA.md)
Comprehensive database schema documentation:
- All database tables and their structures
- Relationships between tables
- Constraints and indexes
- Data integrity rules
- Migration strategy

**Essential for** database agents and backend developers.

### [SETUP.md](./SETUP.md)
Setup and deployment guide:
- Local development setup
- Database hosting options (Supabase, Neon, Railway, etc.)
- Netlify deployment instructions
- Email service configuration
- Environment variables
- Security considerations
- Cost estimation

**Use this** when setting up the development environment or deploying to production.

### [CHECKLISTS.md](./CHECKLISTS.md)
Role-specific implementation checklists:
- **Database Agent Checklist**: Schema, migrations, seed data
- **SvelteKit Pro Agent Checklist**: API routes, server logic, business rules
- **Frontend UI Agent Checklist**: Pages, components, UX, accessibility
- **Documentation Agent Checklist**: Keeping docs up-to-date

**Track progress** by checking off items as you complete them.

### [AGENTS.md](./AGENTS.md)
Agent guide and role definitions:
- Agent roles and their responsibilities
- Role-specific resources and documentation
- Project structure overview
- Key technologies and patterns
- Base prompt template for continuing checklist work

**Essential for** AI agents and developers working on the project. Use this to understand which resources are relevant to your role.

## Quick Start

1. **New to the project?** Start with [SPECIFICATION.md](./SPECIFICATION.md)
2. **AI Agent or Developer?** Read [AGENTS.md](./AGENTS.md) to understand your role and resources
3. **Setting up locally?** Follow [SETUP.md](./SETUP.md)
4. **Implementing features?** Use [CHECKLISTS.md](./CHECKLISTS.md) to track your work
5. **Working with the database?** Reference [SCHEMA.md](./SCHEMA.md)
